fetch

fetch() 是 JavaScript 中用于发起网络请求的现代 API，替代了传统的 XMLHttpRequest，提供了更简洁、更强大的 Promise 式接口

基本语法

javascript

参数说明

  1. **url**（必选）：请求的 URL。
  2. **options**（可选）：配置对象，常用属性：
    • **method**：HTTP 方法（GET、POST、PUT、DELETE 等）。
    • **headers**：请求头对象（如：{'Content-Type': 'application/json'}）。
    • **body**：请求体（通常用于 POST/PUT，需为字符串或 FormData）。
    • **mode**：请求模式（如：cors、no-cors、same-origin）。
    • **credentials**：是否携带 cookies（omit、same-origin、include）。

处理响应数据

fetch 的响应对象（response）提供多种解析方法（均返回 Promise）：

  • **response.json()**：解析为 JSON 对象。
  • **response.text()**：解析为文本。
  • **response.blob()**：解析为二进制文件（如图片、视频）。
  • **response.formData()**：解析为 FormData 对象。
  • **response.arrayBuffer()**：解析为二进制数据缓冲区。

错误处理

  • 网络错误（如断网、DNS 失败）：会触发 .catch()。

  • HTTP 错误（如 404、500）：**不会触发 .catch()**，需手动检查 response.ok（状态码 200-299）。

常见注意事项

  1. CORS 限制：跨域请求需服务器配置 Access-Control-Allow-* 头。
  2. 响应体只能解析一次：多次调用 response.json() 或其他解析方法会报错。
  3. 默认不发送 cookies：需显式设置 credentials: 'include'。
  4. 表单提交：使用 FormData 时无需手动设置 Content-Type。

替代方案

  • axios：功能更全面的 HTTP 客户端（自动处理 JSON 解析、错误处理更友好）。
  • 原生 fetch：适合轻量级场景，无需额外依赖。

问题描述

fetch(“./1.txt”).then((response)=>{ console.log(response.text()); // return response.text() }) 为什么这样读取不成功 要在加一层then()方法呢

1. response.text() 返回的是一个 Promise

  • fetch 返回的 response 对象的方法（如 text()、json()、blob()）不会立即返回数据，而是返回一个 Promise 对象，表示 “未来会得到文本数据”。

  • 直接打印 response.text() 会输出一个 未完成的 Promise，而不是实际的文本内容。

    javascript

2. 异步操作的执行顺序

  • JavaScript 是单线程语言，then() 中的代码会在 主线程同步代码执行完毕后 才会执行（异步回调机制）。
  • response.text() 的 Promise 需要时间解析数据，直接打印时数据尚未就绪。

二、为什么需要再嵌套一层 then()？

1. Promise 链式调用的本质

  • fetch 返回的是一个 Promise，调用第一个 then() 后，response.text() 返回的 新 Promise 需要通过 下一个 then() 来捕获结果。
  • 每一层 then() 都对应一个异步操作的 阶段：
    1. 第一个 then()：处理 fetch 请求成功后的 response 对象。
    2. 第二个 then()：处理 response.text() 解析成功后的文本数据。

2. 正确写法：链式调用处理多层 Promise

三、关键概念：Promise 的链式调用规则

  1. 每一步返回值决定下一步处理逻辑：

    • 如果 then() 中返回 普通值，下一个 then() 会收到该值（会被包装为 Promise.resolve(值)）。
    • 如果 then() 中返回 Promise（如 response.text()），下一个 then() 会等待该 Promise resolved 后再执行。

  2. 避免丢失 Promise：

    • 如果在第一个 then() 中 **没有返回 response.text()**（即没有返回 Promise），第二个 then() 会提前拿到 undefined，导致数据丢失。

    javascript

五、常见错误总结

错误写法 问题原因 正确写法
console.log(response.text()) 直接打印 Promise，未等待解析 用 then() 或 await 处理 Promise
未返回 response.text() 丢失 Promise，导致下一层拿不到数据 在 then() 中返回 response.text()
多层嵌套不规范 代码可读性差，容易出错 使用 async/await 或规范的链式调用

总结

  • fetch 和 response.text() 都是异步操作，会返回 Promise，需要通过 链式 then() 或 async/await 逐层处理。
  • 核心原则：每个 Promise 都需要用 then() 或 await 来获取结果，不能直接打印 Promise 对象。
